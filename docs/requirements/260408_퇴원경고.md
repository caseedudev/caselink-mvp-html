## I. 시스템 개요 및 핵심 알고리즘 목록
CaseLink는 사교육 환경에서 강사·학생·학부모 간의 학습 데이터를 통합 관리하는 학원 운영 플랫폼입니다. 단순한 **"기록 도구"**가 아닌 **"개입(Intervention) 도구"**를 핵심 철학으로 하며, 학원의 표준 운영 매뉴얼을 디지털 알고리즘화하여 강사가 판단 없이도 매뉴얼대로 행동할 수 있도록 지원합니다.

### 핵심 알고리즘 목록
| No | 알고리즘명 | 특허 핵심 가치 |
| :-- | :--- | :--- |
| 1 | 재원생 학교별 내신 평균 자동 산출 | 외부 데이터 없이 재원생 데이터만으로 학교별 평균 자동 계산 |
| 2 | 학부모 성향 자동 분류 및 상담 전략 도출 | 상담 이력 텍스트 분석 → 성향 태그 + 맞춤 상담 전략 자동 생성 |
| 3 | 내신대비 단계 프로세스 마스터 구조 | 반 단위 1회 설정으로 전체 학생 자동 배정 및 이력 누적 |
| 4 | 현재 날짜 기반 내신 시험 자동 예측 | 현재 날짜 분석 → 학기·시험 종류·시험일 자동 추천 |
| 5 | 모의고사 점수 추이 분석 및 색상 코딩 | 회차별 점수 변화 추세 자동 판정 + 5단계 시각적 색상 인코딩 |
| 6 | 학생부-수업관리 데이터 동기화 | 단일 마스터 저장소에서 두 화면이 실시간 동기화되는 구조 |
| 7 | 퇴원 위험 학생 자동 감지 | 출결·점수·상담 이력 복합 분석 → 위험도 분류 및 계층 알림 |
| 8 | 학습 보고서 자동 생성 (Hybrid) | LLM API + 규칙 기반 이중 엔진으로 안정성·품질 동시 확보 |
| 9 | 신입생 3단계 상담 루틴 (신규) | 등록일 기준 D+1·D+7·D+30 자동 계산 및 체크리스트 제시 |
| 10 | 다차원 위험 경고 메시지 (신규) | 4개 지표 복합 분석 → 등급별 경고 메시지 자동 생성 |
| 11 | 예상퇴원생 관리 시스템 (신규) | 퇴원 예측 → 상담 분류 → 보고서 자동 완성 전체 파이프라인 |
| 12 | 일일 업무 흐름 순서화 (신규) | 업무 종속성 기반 4단계 흐름 자동 구성 및 우선순위화 |

## II. 알고리즘 1 — 재원생 학교별 내신 평균 자동 산출

### 1-1. 기술적 배경
기존 시스템은 개인 성적만 기록할 뿐, 학교 공식 평균이 공개되기 전까지는 실시간 비교가 불가능했습니다. CaseLink는 보유한 재원생 데이터를 활용해 실시간 비교 지표를 산출합니다.

### 1-2. 알고리즘 구조
* **함수명:** `calcSchoolAvg(school, exam)`
* **입력:** 학교명(string), 시험명(string)
* **로직:** 전체 재원생 데이터 중 조건에 부합하는 점수들의 산술 평균(반올림) 반환.
* **시간복잡도:** $O(N \cdot M)$ ($N$: 전체 학생 수, $M$: 학생당 시험 기록 수)

### 1-3. 특허 핵심 클레임
* 외부 데이터 없이 학원 자체 데이터만으로 실시간 학교별 평균을 산출하는 방법.
* 성적 입력 시 동일 학교 재원생 평균이 자동 갱신되어 관련 학생 지표에 즉시 반영되는 구조.

---

## III. 알고리즘 2 — 학부모 성향 자동 분류 및 상담 전략 도출

### 2-1. 하이브리드 처리 구조
1.  **1단계:** LLM API(Claude API) 호출 — 상담 이력을 컨텍스트로 제공하여 구조화 JSON 획득.
2.  **2단계:** 규칙 기반 폴백 엔진 — API 실패 시 키워드 분석으로 자동 전환.

### 2-2. 규칙 기반 분류 로직
| 분류 조건 | 도출 성향 태그 | 맞춤 전략 |
| :--- | :--- | :--- |
| "퇴원" 포함 | 퇴원 위험 | 공감 우선 → 조건부 유지 제안 |
| "과제" 포함 | 과제 민감 | 과제 부담 경감 옵션 선제 제시 |
| "점수/성적" 포함 | 성과 중시 | 정기 성취 데이터 공유 강화 |
| 상담 3건 이상 | 적극 소통형 | 신속 회신 및 상세 보고 제공 |

### 2-3. 특허 핵심 클레임
* 자연어 상담 텍스트를 분석하여 학부모를 성향 카테고리로 자동 분류하는 방법.
* LLM과 규칙 기반 엔진의 하이브리드 구성을 통한 서비스 연속성 보장.

## IV. 알고리즘 3 — 내신대비 단계 프로세스 마스터 구조

### 3-1. 데이터 구조
* `nsMasterProcesses[]`: 시험별 프로세스 메타데이터 (시험명, 반, 단계별 과제 등)
* `nsMasterChecks{procKey}{studentId}`: 학생별 수행 데이터 (완료 여부, 점수 등)

### 3-2. 특허 핵심 클레임
* 반 단위 1회 설정으로 전체 학생에게 자동 배정되는 메커니즘.
* 메타데이터와 수행 데이터를 분리 저장하여 데이터 효율성 및 이력 누적 기능을 강화한 구조.

## V. 알고리즘 4 — 현재 날짜 기반 내신 시험 자동 예측

### 4-1. 날짜 분류 및 추천 규칙
| 현재 월 | 학기/시험 종류 | 추천 시험일 | 추천 대비 시작일 |

| 1월~5월 | 1학기 중간고사 | 5월 15일 | 4월 17일 |

| 11월~12월 | 2학기 기말고사 | 12월 10일 | 11월 13일 |

### 4-2. 특허 핵심 클레임
* 한국 학사 일정 패턴을 추론하여 시험 종류와 일정을 자동 예측하는 방법.
* 자동 생성된 값을 수정 가능한 **Editable Auto-fill UX** 설계.

## VI. 알고리즘 5 — 모의고사 점수 추이 분석 및 색상 코딩

### 5-1. 추세 판정 및 구간 색상
* **상승:** 마지막-첫 점수 $\ge$ +5점 (**Teal**)
* **안정:** -5점 < 변화 < +5점 (**Blue**)
* **하락:** 마지막-첫 점수 $\le$ -5점 (**Red**)

| 점수 범위 | 색상 | 의미 |
| :--- | :--- | :--- |
| 85점 이상 | Teal | 우수 (목표 달성) |
| 70~84점 | Blue | 양호 (안정권) |
| 55~69점 | Amber | 주의 (보완 필요) |
| 54점 이하 | Red | 위험 (집중 지도) |

## VII. 알고리즘 6 — 학생부-수업관리 양방향 데이터 동기화
* **구조:** 반 단위 입력 화면(수업관리)과 학생 개인 조회 화면(학생부)이 단일 마스터를 공유.
* **핵심:** 이중 뷰 아키텍처를 통해 데이터 일관성을 보장하고 각 관점에 맞는 렌더링 제공.

---

## VIII. 알고리즘 7 — 퇴원 위험 학생 자동 감지 및 상담 트리거
* **판단 기준:** 출결률 85% 미만, 테스트 점수 2회 하락, 과제 미흡 3회, 특정 키워드(퇴원 등).
* **알림 트리거:** 위험도에 따라 강사 알림 또는 부서장 알림으로 자동 분기.

---

## IX. 알고리즘 8 — 학습 보고서 자동 생성 (Hybrid)
* **로직:** LLM API를 1차로 사용하고, 실패 시 미리 정의된 임계값 규칙(출결, 점수, 과제 등)에 따라 서술형 문장을 생성.
* **특징:** 카카오톡 발송 포맷에 최적화된 문체 자동 생성.

---

## X. 알고리즘 9 — 신입생 등록일 기반 3단계 상담 루틴 자동 트리거 ★ 신규

### 10-1. 3단계 루틴 구조
1.  **1단계 (D+1):** 첫날 상담 (시스템 안내, 첫인상 피드백)
2.  **2단계 (D+7):** 1주일 상담 (적응 및 과제 수행 확인)
3.  **3단계 (D+30):** 1개월 상담 (학습 개선점 및 적응 상황 종합 점검)

### 10-2. 특허 핵심 클레임
* 등록일 기준 상담 시점 자동 계산 및 단계별 체크리스트 자동 제시 방법.
* 신입생 적응도(4단계) 분류에 따른 계층적 에스컬레이션(부서장 알림) 구조.

---

## XI. 알고리즘 10 — 다차원 위험 경고 메시지 자동 생성 ★ 신규

### 11-1. 경고 등급 분류
* **🔴 위험:** 퇴원 의사 표명 등 (빨간 뱃지 + 진동 + 부서장 알림)
* **🟠 주의:** 출결/점수 하락 (주황 뱃지 + 강사 팝업)
* **🔵 관찰:** 상담 미완료 등 (파란 뱃지 + 업무 목록 추가)

### 11-2. 특허 핵심 클레임
* 데이터 실시간 모니터링을 통한 3단계 위험도 자동 분류.
* 지표값을 템플릿에 동적 삽입하여 즉각 대응 가능한 구체적 메시지 생성 방법.

---

## XII. 알고리즘 11 — 예상퇴원생 관리 시스템 알고리즘 ★ 신규

### 12-1. 관리 파이프라인
* **감지:** 복합 지표 분석 → **상담 분기:** 유형별 자동 분류 → **문서 완성:** 학습 데이터 자동 매핑 → **승인:** 부서장 최종 결재.

### 12-2. 특허 핵심 클레임
* 퇴원 위험 감지부터 승인까지의 통합 파이프라인 프로세스.
* 학습 이력(출결, 점수, 상담 등)을 퇴원서 양식에 자동으로 취합 및 입력하는 방법.

---

## XIII. 알고리즘 12 — 일일 업무 흐름 자동 순서화 알고리즘 ★ 신규

### 13-1. 4단계 업무 종속성
* **STEP 1:** 수업 기록 입력 (시간 의존)
* **STEP 2:** 상담 입력 (수업 기록 일부 완료 시 가능)
* **STEP 3:** 알림톡 발송 (수업 기록 완료 필수 - 강한 의존)
* **STEP 4:** 업무일지 제출 (이전 단계 완료 권장)

### 13-2. 상담 우선순위
1.  예상퇴원/긴급상담
2.  콜백 요청
3.  신입생 단계 상담
4.  재원생 정기 상담

### 13-3. 특허 핵심 클레임
